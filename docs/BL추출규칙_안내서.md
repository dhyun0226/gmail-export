# 메일 BL 추출 규칙 및 제외 규칙 안내서

## 개요

이 문서는 Gmail에서 수입 관련 메일을 읽어 BL(선하증권) 번호를 자동으로 추출하는 규칙과,
처리 대상에서 제외하는 메일 필터 규칙을 정리한 문서입니다.


---


## 1. BL 번호 추출 규칙

메일 제목(Subject)에서 아래 패턴을 **위에서부터 순서대로** 매칭합니다.
먼저 매칭되는 규칙이 있으면 그 결과를 BL 번호로 사용하고, 아래 규칙은 건너뜁니다.

### 1-1. 패턴 매칭 (제목 기준)

| 순번 | 규칙 이름 | 매칭 조건 | 추출 대상 | 예시 |
|:---:|-----------|-----------|-----------|------|
| 1 | 항공수입 # | 제목에 `항공수입 #` 뒤에 코드가 있을 때 | # 뒤의 코드 | `항공수입 #ABC123` → **ABC123** |
| 2 | HB: | 제목에 `HB:` 뒤에 숫자가 있을 때 | 숫자 부분 | `HB: 98765` → **98765** |
| 3 | HAWB# | 제목에 `HAWB#` 뒤에 코드가 있을 때 | 코드 부분 | `HAWB# XY789` → **XY789** |
| 4 | HAWB: | 제목에 `HAWB:` 뒤에 코드가 있을 때 | 코드 부분 | `HAWB: XY789` → **XY789** |
| 5 | FedEx AWB | 제목에 `AWB # -` 뒤에 숫자가 있을 때 | 숫자 부분 | `AWB # - 7890123` → **7890123** |
| 6 | DHL | 제목에 `DHL` 뒤에 숫자가 있을 때 | 숫자 부분 | `DHL 1234567890` → **1234567890** |
| 7 | DGF 답장 | 영문 1글자 + 숫자 6자리 이상 단어가 있을 때 | 해당 단어 | `Re: H687676 수입건` → **H687676** |
| 8 | CNEE/CNW # | `++CNEE++ CNW #` 뒤에 숫자가 있을 때 | 숫자 부분 | `++CNEE++ CNW # 55555` → **55555** |
| 9 | CNW Pre-Alert | `CNW Pre-Alert -` 뒤에 숫자가 있을 때 | 숫자 부분 | `CNW Pre-Alert - 55555` → **55555** |
| 10 | CEVA Shipment | `Shipment No` 뒤에 코드가 있을 때 | 코드 부분 | `Shipment No SH001` → **SH001** |
| 11 | CEVA HAWB N° | `HAWB N°` 뒤에 코드가 있을 때 | 코드 부분 | `HAWB N° AB1234` → **AB1234** |
| 12 | AWB (범용) | 제목에 `AWB` 뒤에 코드가 있을 때 | 코드 부분 | `AWB 12345` → **12345** |

### 1-2. 보조 규칙 (패턴 매칭 실패 시)

위 12개 패턴에 매칭되지 않으면 아래 보조 규칙을 순서대로 시도합니다.

| 순번 | 조건 | 동작 | 예시 |
|:---:|------|------|------|
| A | 보낸 사람이 `@dhl.com`이고, 제목 첫 단어가 `영문1자+숫자` 형태 | 첫 단어를 BL로 추출 | 보낸사람: noreply@dhl.com / 제목: `H123456 배송건` → **H123456** |
| B | 제목에 `/`(슬래시)가 포함되어 있고, `/` 앞부분이 `영문1자+숫자` 형태 | `/` 앞부분을 BL로 추출 | `H123456/수입통관` → **H123456** |

### 1-3. UPS 메일 전용 규칙

UPS 메일은 별도 로직으로 처리합니다.

- **전제조건**: 제목에 `[Pre-Alert]`이 포함된 메일만 처리
- **본문에서 추출하는 정보**:
  - BL 번호: `BL Number:`, `B/L`, `BL` 라벨 뒤의 10자리 이상 코드
  - 트래킹 번호: `Tracking Number:`, `Track` 라벨 뒤의 코드
- **트래킹 번호 판별 기준**: `1Z`로 시작하거나 숫자 12자리
- **BL 번호 판별 기준**: 영숫자 10~11자리이면서 `1Z`로 시작하지 않는 것

### 1-4. 추출 실패 시

위 모든 규칙에 매칭되지 않으면 BL 번호는 `N/A`로 표시됩니다.

## 2. 메일 제외(필터) 규칙

아래 조건에 해당하는 메일은 BL 추출 대상에서 **제외**됩니다.

### 2-1. 제외 키워드

메일 제목에 아래 키워드가 **포함**되어 있으면 제외합니다.

| 번호 | 키워드 | 설명 |
|:---:|--------|------|
| 1 | 순중량 정정요청 | 중량 정정 관련 |
| 2 | 미결안내/용도설명서 요청 | 미결/서류 요청 |
| 3 | PENDING LIST | 미결 리스트 |
| 4 | 미반출 내역 안내의 건 | 미반출 안내 |
| 5 | Microsoft OneDrive 확인 코드 | 시스템 인증 메일 |
| 6 | 운송리스트 [TOP Customs] | 운송리스트 안내 |
| 7 | check in Korea HTS | HTS 확인 요청 |
| 8 | BOM 전달 | BOM 자료 전달 |
| 9 | 가산금적용여부문의 | 가산금 문의 |
| 10 | 이고요청 | 이고 요청 |
| 11 | 중량문의 | 중량 관련 문의 |
| 12 | 중량 문의 | 중량 관련 문의 (띄어쓰기 포함) |
| 13 | 통관 지연 사유 문의 | 통관 지연 문의 |
| 14 | 수입신고필증 전달건 | 신고필증 전달 |
| 15 | 입항통보건 리스트 안내 | 입항 리스트 안내 |
| 16 | 입고요청 | 입고 요청 |
| 17 | RATED B/L 요청 | Rated BL 요청 |
| 18 | 스케줄 지연 안내 | 스케줄 안내 |
| 19 | 원산지 확인요청 | 원산지 확인 |

### 2-2. 답장(RE:) 메일 필터

- `RE:`로 시작하는 답장 메일은 **기본적으로 제외**합니다.
- **예외**: 제목이 `RE: ++CNEE++ ... CNW Pre-Alert` 형태인 답장은 제외하지 않고 처리합니다.


---


## 3. 처리 흐름 요약

```
메일 수신
  │
  ├─ 제목에 제외 키워드 포함? ──── Yes ──→ 제외 (처리 안 함)
  │
  ├─ RE: 답장 메일? ──── Yes ──→ CNW Pre-Alert 답장? ── No ──→ 제외
  │                                       │
  │                                      Yes
  │                                       │
  ├───────────────────────────────────────┘
  │
  ▼
BL 추출 시도
  │
  ├─ 패턴 1~12 매칭 시도 (순서대로)
  ├─ 보조 규칙 A, B 시도
  ├─ UPS [Pre-Alert] 별도 처리
  │
  ▼
결과: BL 번호 또는 N/A
```

## 4. 참고

- 소스 파일 위치
  - BL 추출: `server/utils/blExtractor.ts`
  - 메일 필터: `server/utils/emailFilter.ts`
- 패턴은 위에서 아래로 우선순위가 높은 순서입니다.
  더 구체적인 패턴(예: HAWB#)이 먼저 매칭되고, 범용 패턴(예: AWB)은 나중에 시도됩니다.
